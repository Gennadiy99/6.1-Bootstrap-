Как Правильно писать медиазапросы для адаптивной верстки

Медиазапросы пишутся в конце всех стилей или в отдельном файле и подключаются после своего файла "CSS"

Помним о принципе в "CSS" те свойства, что указаны для элемента изначально, применяется и далее если их не переопределить.

В файле "CSS" нужно указывать стили экрана от большего к меньшему.

Что бы управлять адаптивностью элементов через классы, нужно создать классы, в медиазапросе и они будут работать только при этой ширине экрана. 

            @media screen and (max-width: 768px) {
              .col-md-3 {
                width: calc(25% - var(--gutter));
              }
              .col-md-4 {
                width: calc(33.33% - var(--gutter));
              }
              .col-md-6 {
                width: calc(50% - var(--gutter));
              }
              .col-md-12 {
                width: calc(100% - var(--gutter));
              }
            } 
            Теперь эти классы можно применят к элементам и они будут срабатывать только при указанной в "Медиа-запросе",- ширине экрана.

Так же нужно помнить, что можно использовать отделные элементы из "Bootstrap" и скачивать то что вы хотите использовать, к примеру можно скачать только сетку Bootstrap и использовать только ее.

  • Для этого нужно загрузить с сайта Bootstrap весь каталог
  • Зайти папку "dist"
  • Потом в папку "css"
  • И найти файл bootstrap-grid-min.css
  • Скопировать его и перенести его к себе в папку "css"

7.6 Самостоятельная верстка

Очень важный момент. При наполнении блока Фоновым изображением нужно обязательно задавать только фиксируемую высоту, Иначе изображение не отобразиться или нужно указать "padding-top:20%" тогда Изображение отобразится. Важно что при помощи "padding-top: ... %" можно влиять на соотношение сторон фонового изображения!!! то есть делать его уже или шире по высоте.

При установке иконок из Fontello в тег "i" при помощи классов, устанавливается в стилях псевдоэлемент "before" в котором есть боковые отступы "margin" если они влияют на на верстку, из за них выпадает иконка из блока или влияет "margin" на ширину блока, то переопределить стиль можно увеличив значимость "Селектора" добавив к i[class^="icon-"]:before{ margin: 0; } тег "a" в котором находится "i" a > i[class^= icon-]:before { margin: 0; } Тогда получится переопределить отступ "margin"

Что бы при Верстке блоки с динамическим размером (%) и фиксированным отступом "margin-bottom" не отодвигали кнопки за приделы общего блока, НУЖНО использовать функцию calc() для свойства height: height: calc(100% - 140px) где 140px это тот зазор который будет всегда вычитаться из высоты блока (относительно текущего размера) и резирвироваться для элементов, - кнопок, их размеров и отступов. Конечно в каждом случаи количество "px" будет свое, относительно размеров и зазоров у элементов находящихся в блоке.

Важное примечание! Иногда при настройке адапривности (малых экранов) устанавливается для элементов свойство: max-width: 280px; И элемент уменьшается, не вылазит за пределы Контейнера.
При этом на больших экранах, эти флекс элементы ведут себя "Некорректно" в размещении, по этому задается дополнительно этим элементам свойсто: width: 100%; 

              .item {
                
                  max-width: 280px;
                  width: 100%;
                
                height: 120px;
                text-align: center;
                display: flex;
                flex-direction: column;
                justify-content: space-between;
              }
